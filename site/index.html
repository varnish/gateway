Why Varnish Gateway?

All the routing capabilities of the Gateway API, with the speed and flexibility of Varnish.

Conformance Tested

Passes the Gateway API conformance test suite. Standard HTTPRoute matching, header and query filters, path types, and traffic splitting.

Zero VCL Generation

HTTP routing is not driven by VCL. Instead, the Varnish Gateway routes based on live, in-memory routing tables derived from Gateway API HTTPRoutes and Kubernetes EndpointSlices. Route and backend changes require no VCL regeneration, making updates fast and operationally safe.

Built-in Dashboard

Real-time monitoring dashboard with live event stream, backend health, virtual host overview, and heartbeat visualization.

Custom VCL

HTTPRoutes handle standard routing and Gateway API features. When you need something beyond the spec, VCL is there for custom caching, rewriting, or request logic.

Multi-Listener

Multiple Gateway listeners on different ports. Listener-aware routing with per-listener VCL branching via request headers.

TLS Termination

Native, high-performance TLS support on both client and backend side. Certificates hot-reload automatically with no restarts.

Gateway API Conformance

Varnish Gateway passes the official Kubernetes Gateway API conformance test suite, covering HTTPRoute matching, request/response header filters, URL rewrites, redirects, and weighted traffic splitting.

How it works

Three components work together to translate Gateway API resources into a running Varnish configuration.

1

Operator

Watches Gateway and HTTPRoute resources. Generates VCL and routing configuration. Manages the Varnish Deployment, Service, and RBAC.

2

Chaperone

Runs as a sidecar managing the varnishd process. Discovers endpoints via EndpointSlices and merges them with routing config into ghost.json.

3

Ghost VMOD

A Rust-based Varnish module that handles all routing inside Varnish. Reads ghost.json, matches requests to backends by host, path, headers, and query parameters.

Getting started

Install Varnish Gateway in your Kubernetes cluster using Helm or plain manifests.

Install the Gateway API CRDs:

kubectl apply -f https://github.com/kubernetes-sigs/gateway-api/releases/download/v1.2.0/standard-install.yaml

Install Varnish Gateway with Helm:

helm install varnish-gateway ./charts/varnish-gateway \ --namespace varnish-gateway-system \ --create-namespace

Create a Gateway:

kubectl apply -f - <<EOF apiVersion: gateway.networking.k8s.io/v1 kind: Gateway metadata: name: varnish spec: gatewayClassName: varnish listeners: - name: http port: 80 protocol: HTTP EOF

Install the Gateway API CRDs:

kubectl apply -f https://github.com/kubernetes-sigs/gateway-api/releases/download/v1.2.0/standard-install.yaml

Deploy the operator and resources:

kubectl apply -f deploy/
